## Key cross-cutting things to know (top 10)

If you're new to this codebase, internalize these before making changes:

1. **Two HTTP surfaces share models and services.** Inertia for auth/dashboard/settings/landing/checkout shells; REST `/api/v1/*` for catalog refresh, cart, orders, payments, admin CRUD. See `architecture.md §3`.
2. **Cart endpoints live under `/api/v1/cart` but use the `web` middleware** (so they get session + CSRF). The frontend additionally sends `X-Session-ID` to keep guest carts stable across CSRF resets.
3. **Address book endpoints are declared in `routes/web.php`** (not `routes/api.php`) under `auth` + `verified` middleware — session auth despite the `/api/v1/user/addresses` URL. Documented in `BACKEND_DISCREPANCIES.md`.
4. **Business logic lives in `app/Services/`** (`CartService`, `OrderService`, `PayPalPaymentService`). Controllers stay thin. Always wrap multi-table mutations in `DB::transaction`.
5. **`StoreOrderRequest` branches its validation rules** based on `Auth::guard('sanctum')->check() || Auth::guard('web')->check()`. Authenticated users send address ids; guests embed full address objects.
6. **Stock is decremented at order creation, restored on cancel.** No separate "reservation" table — order cancellation is the only recovery mechanism for held stock.
7. **`PAYPAL_SIMULATE_PAYMENTS=true` (or `APP_ENV=testing`) routes payments through a simulator.** Real PayPal calls go through `PayPalPaymentService` (currency `MXN`, capture flow).
8. **PayPal payment endpoints are currently public.** They validate the order's `payment_status` is `pendiente` but don't check ownership. Tighten before going live.
9. **Order confirmation email is queued (`ShouldQueue`)** — production needs a `queue:work` worker or emails won't leave.
10. **Use the `Makefile`.** It abstracts over `docker compose exec`, `APP_ENV=testing`, and the right Cypress config. In Docker, run E2E via `make test-e2e`, not `npm run test:e2e`.
11. **Tests use a separate MariaDB (`db_test`)**, not the dev DB. PHPUnit hits it via env vars in `phpunit.xml`. Cypress E2E temporarily swaps the dev container's `.env` to point to `cronosmatic_test`, runs, then restores. See `docs/development-guide.md §How the isolation works`.
